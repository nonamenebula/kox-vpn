## Сценарий Б — Удалить Kvass и настроить VLESS

### Б-1. Подключиться к Entware shell по SSH

```bash
ssh root@192.168.1.1 -p 222
# пароль — тот же, что у веб-интерфейса (или "keenetic" по умолчанию)
```

### Б-2. Остановить и отключить Kvass

```bash
# Остановить сервис
/opt/etc/init.d/S96kvas stop 2>/dev/null || true

# Удалить пакет
/opt/bin/opkg remove kvas

# Удалить оставшиеся файлы
rm -f /opt/etc/init.d/S96kvas
rm -f /opt/etc/ndm/iflayerchanged.d/kvas-ips-reset
rm -f /opt/etc/ndm/ifdestroyed.d/kvas-iface-del
rm -f /opt/etc/ndm/ifcreated.d/kvas-iface-add
rm -f /opt/etc/cron.5mins/ipset.kvas
rm -f /opt/etc/dnsmasq.d/kvas.dnsmasq
rm -f /opt/etc/kvas.dnsmasq
rm -f /opt/bin/kvas
rm -rf /opt/apps/kvas /opt/etc/.kvas
rm -f /opt/etc/kvas.conf /opt/etc/kvas.list
```

### Б-3. Удалить старые правила iptables от Kvass

```bash
iptables -t nat -D PREROUTING -i br0 -p tcp -m set --match-set unblock dst -j REDIRECT --to-ports 1181 2>/dev/null
iptables -t nat -D PREROUTING -i br0 -p udp -m set --match-set unblock dst -j REDIRECT --to-ports 1181 2>/dev/null
```

### Б-4. Если был shadowsocks — отключить его

```bash
sed -i 's/^ENABLED=yes/ENABLED=no/' /opt/etc/init.d/S22shadowsocks
/opt/etc/init.d/S22shadowsocks stop 2>/dev/null || true
killall ss-redir 2>/dev/null || true
```

### Б-5. Продолжить с шага «Установка Xray»

## Установка Entware

> Пропустите этот раздел, если Entware уже установлен (проверьте: `ls /opt/bin/opkg`).

### Ш1. Включить компонент OPKG в веб-интерфейсе

1. Откройте **http://192.168.1.1**
2. **Управление → Обновление KeeneticOS → Параметры компонентов**
3. Найдите **«Поддержка пакетов OPKG»** → нажмите **Установить**
4. Дождитесь перезагрузки

### Ш2. Подготовить USB-флешку

Отформатируйте флешку в **EXT4** (Linux/macOS: `mkfs.ext4 /dev/sdX`) или **NTFS**.  
На Mac можно использовать Disk Utility → формат MS-DOS/ExFAT, затем роутер сам отформатирует внутри.

### Ш3. Подключить флешку и установить Entware

1. Вставьте флешку в USB роутера
2. Откройте веб-интерфейс → **Приложения → Entware** (или раздел OPKG)
3. Нажмите **«Установить»** напротив нужного раздела диска
4. Либо через CLI NDMS:
```
(config)> opkg disk EXT_LABEL:
(config)> system configuration save
```
Где `EXT_LABEL` — метка вашего раздела.

### Ш4. Убедиться что Entware работает

```bash
ssh root@192.168.1.1 -p 222
# введите пароль
ls /opt/bin/   # должны быть файлы
/opt/bin/opkg update
```

## Установка Xray

### Ш1. Установить пакет

```bash
ssh root@192.168.1.1 -p 222

/opt/bin/opkg update
/opt/bin/opkg install xray-core
# ИЛИ если не найден:
/opt/bin/opkg install xray
```

> На MIPS-роутерах Keenetic пакет называется **`xray-core`** (предоставляет бинарник `/opt/sbin/xray`).  
> На других архитектурах (ARM) — тот же пакет.

### Ш2. Скачать geoip.dat и geosite.dat

Эти файлы нужны для правила `geoip:private` (не направлять локальный трафик в VPN).

**Вариант А — с компьютера в той же сети (рекомендуется):**

На вашем Mac/ПК:
```bash
# Скачать файлы
curl -L -o /tmp/geoip.dat \
  "https://cdn.jsdelivr.net/gh/Loyalsoldier/v2ray-rules-dat@release/geoip.dat"
curl -L -o /tmp/geosite.dat \
  "https://cdn.jsdelivr.net/gh/Loyalsoldier/v2ray-rules-dat@release/geosite.dat"

# Запустить временный HTTP-сервер
# Узнайте свой IP в сети: ifconfig | grep 192.168
cd /tmp && python3 -m http.server 8765 --bind 192.168.1.ВАШ_IP
```

На роутере (через SSH порт 222):
```bash
mkdir -p /opt/usr/share/xray
wget -O /opt/usr/share/xray/geoip.dat    http://192.168.1.ВАШ_IP:8765/geoip.dat
wget -O /opt/usr/share/xray/geosite.dat  http://192.168.1.ВАШ_IP:8765/geosite.dat
```

**Создать симлинки** (Xray ищет .dat рядом с бинарником):
```bash
ln -sf /opt/usr/share/xray/geoip.dat   /opt/sbin/geoip.dat
ln -sf /opt/usr/share/xray/geosite.dat /opt/sbin/geosite.dat
```

## Создание конфига Xray

### Ш1. Получить данные из вашего VLESS URI

Ваш URI выглядит так:
```
vless://UUID@HOST:PORT?type=tcp&security=reality&sni=SNI&fp=FINGERPRINT&pbk=PUBLIC_KEY&sid=SHORT_ID&flow=xtls-rprx-vision#НАЗВАНИЕ
```

Запишите:
- `UUID` — уникальный идентификатор пользователя
- `HOST` — адрес сервера
- `PORT` — порт (обычно 443)
- `SNI` — server name (обычно = HOST)
- `FINGERPRINT` — `chrome` или другой
- `PUBLIC_KEY` — публичный ключ Reality
- `SHORT_ID` — короткий ID

## Скрипты iptables

Эти скрипты перенаправляют LAN-трафик в Xray (прозрачный прокси).

### Ш1. Создать скрипт netfilter (вызывается при пересборке фаервола)

```bash
cat > /opt/etc/ndm/netfilter.d/99-xray-nat.sh << 'EOF'
#!/bin/sh
# LAN TCP -> Xray dokodemo 10808 (IPv4 + IPv6).
PATH=/opt/sbin:/opt/bin:/sbin:/usr/sbin:/usr/bin:/bin
export PATH

apply_v4() {
  iptables -t nat -D PREROUTING -i br0 -p tcp -j XRAY_REDIRECT 2>/dev/null
  iptables -t nat -D PREROUTING -i br0 -p udp --dport 443 -j XRAY_REDIRECT 2>/dev/null
  iptables -t nat -F XRAY_REDIRECT 2>/dev/null
  iptables -t nat -X XRAY_REDIRECT 2>/dev/null

  iptables -t nat -N XRAY_REDIRECT
  iptables -t nat -A XRAY_REDIRECT -d 0.0.0.0/8    -j RETURN
  iptables -t nat -A XRAY_REDIRECT -d 127.0.0.0/8  -j RETURN
  iptables -t nat -A XRAY_REDIRECT -d 10.0.0.0/8   -j RETURN
  iptables -t nat -A XRAY_REDIRECT -d 172.16.0.0/12 -j RETURN
  iptables -t nat -A XRAY_REDIRECT -d 192.168.0.0/16 -j RETURN
  iptables -t nat -A XRAY_REDIRECT -d 169.254.0.0/16 -j RETURN
  iptables -t nat -A XRAY_REDIRECT -d 224.0.0.0/4  -j RETURN
  iptables -t nat -A XRAY_REDIRECT -p udp --dport 443 -j REDIRECT --to-ports 10808
  iptables -t nat -A XRAY_REDIRECT -p tcp           -j REDIRECT --to-ports 10808
  iptables -t nat -I PREROUTING 1 -i br0 -p tcp           -j XRAY_REDIRECT
  iptables -t nat -I PREROUTING 1 -i br0 -p udp --dport 443 -j XRAY_REDIRECT
}

apply_v6() {
  ip6tables -t nat -D PREROUTING -i br0 -p tcp -j XRAY_REDIRECT 2>/dev/null
  ip6tables -t nat -D PREROUTING -i br0 -p udp --dport 443 -j XRAY_REDIRECT 2>/dev/null
  ip6tables -t nat -F XRAY_REDIRECT 2>/dev/null
  ip6tables -t nat -X XRAY_REDIRECT 2>/dev/null

  ip6tables -t nat -N XRAY_REDIRECT
  ip6tables -t nat -A XRAY_REDIRECT -d ::1/128   -j RETURN
  ip6tables -t nat -A XRAY_REDIRECT -d fe80::/10 -j RETURN
  ip6tables -t nat -A XRAY_REDIRECT -d fc00::/7  -j RETURN
  ip6tables -t nat -A XRAY_REDIRECT -d ff00::/8  -j RETURN
  ip6tables -t nat -A XRAY_REDIRECT -p udp --dport 443 -j REDIRECT --to-ports 10808
  ip6tables -t nat -A XRAY_REDIRECT -p tcp           -j REDIRECT --to-ports 10808
  ip6tables -t nat -I PREROUTING 1 -i br0 -p tcp           -j XRAY_REDIRECT
  ip6tables -t nat -I PREROUTING 1 -i br0 -p udp --dport 443 -j XRAY_REDIRECT
}

if [ -z "$table" ] && [ -z "$type" ]; then
  apply_v4
  apply_v6 2>/dev/null || true
  exit 0
fi

[ "$table" != "nat" ] && exit 0

if [ "$type" = "iptables" ];  then apply_v4; exit 0; fi
if [ "$type" = "ip6tables" ]; then apply_v6 2>/dev/null || true; exit 0; fi
exit 0
EOF

chmod +x /opt/etc/ndm/netfilter.d/99-xray-nat.sh
```

### Ш2. Создать скрипт wan.d (вызывается при поднятии WAN/интернета)

```bash
cat > /opt/etc/ndm/wan.d/99-xray-nat.sh << 'EOF'
#!/bin/sh
# Re-apply Xray NAT after WAN up.
[ "$1" = "start" ] || exit 0
sleep 2
sh /opt/etc/ndm/netfilter.d/99-xray-nat.sh 2>/dev/null
exit 0
EOF

chmod +x /opt/etc/ndm/wan.d/99-xray-nat.sh
```

### Ш3. Применить правила сейчас

```bash
sh /opt/etc/ndm/netfilter.d/99-xray-nat.sh
```

### Ш4. Проверить что правила применились

```bash
iptables -t nat -L XRAY_REDIRECT -n -v
# Должны быть строки REDIRECT -> 10808
```

## Запуск и проверка

### Ш1. Убедиться что S24xray включён

```bash
cat /opt/etc/init.d/S24xray | grep ENABLED
# Должно быть: ENABLED=yes
```

Если `no` — исправить:
```bash
sed -i 's/^ENABLED=no/ENABLED=yes/' /opt/etc/init.d/S24xray
```

### Ш2. Запустить Xray

```bash
/opt/etc/init.d/S24xray start
```

### Ш3. Проверить что Xray слушает порт 10808

```bash
netstat -ln | grep 10808
# Должно быть: tcp ... :::10808 ... LISTEN
```

### Ш4. Проверить соединения с VLESS-сервером

```bash
netstat -tn | grep ':443' | head -10
# Должны быть ESTABLISHED соединения на IP вашего сервера
```

### Ш5. Тест с браузера

1. Откройте **2ip.ru** в браузере на устройстве в Wi-Fi
2. Если IP не совпадает с домашним провайдерским — туннель работает
3. Проверьте YouTube, Instagram — должны открываться
4. Проверьте сайт, которого нет в списке (например gosuslugi.ru) — должен открываться с домашним IP

## Обновление конфига и списка

### Добавить домен в туннель

1. Подключитесь: `ssh root@192.168.1.1 -p 222`
2. Откройте конфиг: `vi /opt/etc/xray/config.json`
3. В секцию `"domain"` добавьте строку: `"domain:newsite.com",`
4. Проверьте: `/opt/sbin/xray run -test -c /opt/etc/xray/config.json`
5. Перезапустите: `/opt/etc/init.d/S24xray restart`

### Исключить домен из туннеля (пустить напрямую)

Просто удалите строку с доменом из секции `"domain"` в конфиге.

### Поменять VLESS-сервер

В конфиге найдите секцию `"outbounds"` → блок `"protocol": "vless"` и замените:
- `"address"` — новый хост
- `"id"` — новый UUID
- `"publicKey"`, `"shortId"`, `"serverName"` — новые параметры Reality

## Советы и устранение проблем

### Трафик идёт не через туннель
```bash
# Проверить что правила iptables на месте
iptables -t nat -L XRAY_REDIRECT -n -v

# Если пустые — применить вручную
sh /opt/etc/ndm/netfilter.d/99-xray-nat.sh
```

### Xray не запускается
```bash
# Проверить лог
tail -20 /opt/var/log/xray-err.log

# Проверить конфиг на ошибки
/opt/sbin/xray run -test -c /opt/etc/xray/config.json
```

### После перезагрузки роутера всё слетело
Проверьте что скрипты `wan.d` и `netfilter.d` существуют и исполняемые:
```bash
ls -la /opt/etc/ndm/wan.d/99-xray-nat.sh
ls -la /opt/etc/ndm/netfilter.d/99-xray-nat.sh
```
Если нет — создайте заново по инструкции выше.

### Включить подробный лог (для диагностики)
В `config.json` измените `"loglevel": "warning"` на `"loglevel": "info"` и добавьте:
```json
"error": "/opt/var/log/xray-err.log",
"access": "/opt/var/log/xray-acc.log"
```
Затем: `/opt/etc/init.d/S24xray restart && tail -f /opt/var/log/xray-err.log`

### На устройстве есть VPN-приложение
Если на телефоне/ноутбуке установлен sing-box, v2rayN, Shadowrocket — **отключите** его при тестировании через роутер. Иначе устройство само строит свой туннель, игнорируя роутер.

### iCloud Private Relay (Apple)
На iPhone/Mac в настройках iCloud может быть включён **Private Relay** — он шифрует весь трафик Apple и обходит роутер. Отключите его для чистого теста: **Настройки → ваше имя → iCloud → Private Relay → Выкл**.

- Для SSH доступа извне используйте **ключи** вместо пароля
